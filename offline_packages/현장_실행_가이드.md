# 현장 실행 가이드 — D-테스트베드

## 순서

### 1단계: 패키지 설치 (처음 1회만)

USB를 현장 PC에 꽂고:

```
offline_packages 폴더에서 install_offline.bat 더블클릭
```

또는 터미널에서:

```cmd
cd USB경로\offline_packages
install_offline.bat
```

---

### 2단계: 데이터 파일 배치

금융결제원에서 제공받은 CSV 파일을 아래 경로에 넣기:

```
model/data/kftc/
    HF_TRNS_TRAN.csv    ← 홈·펌뱅킹 이체 (필수)
    OB_TRNS_TRAN.csv    ← 오픈뱅킹 이체 (필수 — 거래소 링킹)
    CD_TRNS_TRAN.csv    ← ATM 거래
    PI_WD_LEDG.csv      ← 개인 출금 원장 (연령/성별)
    GR_JC_TRAN.csv      ← 지로 자동이체
    CMS_REQ_TRAN.csv    ← CMS 출금
```

**파일이 없어도 됩니다** — 있는 파일만 자동으로 로딩합니다.

---

### 3단계: 컬럼명 확인 (중요!)

현장 데이터 명세서와 아래 파일의 컬럼 매핑을 대조:

```
model/data/real_data_loader.py
```

실제 컬럼명이 다르면 `HF_COL_MAP`, `OB_COL_MAP` 딕셔너리 수정.

예시:
```python
# 명세서에 WD_FC_SN 이라고 되어 있으면
HF_COL_MAP = {
    "WD_FC_SN": "sender_fi",   # ← 이렇게 수정
    ...
}
```

---

### 4단계: 데이터 검증 (5분)

```cmd
cd model
python data/real_data_loader.py
```

출력 예시:
```
[OK] HF_TRNS_TRAN: 1,234,567행 로딩
[OK] OB_TRNS_TRAN: 456,789행 로딩
통합 거래: 1,691,356건
거래소 방향 이체: 12,345건 식별
```

오류 없으면 다음 단계 진행.

---

### 5단계: 전체 파이프라인 실행

```cmd
python main.py --real
```

예상 소요 시간:
- 데이터 로딩: 1~5분
- Layer 1 규칙 엔진: 5~30분 (계좌 수에 따라)
- Layer 2 GNN 학습: GPU 있으면 5분, CPU면 30~60분

---

### 결과 파일

```
model/layer1/layer1_results.csv     ← 규칙 엔진 플래그 결과
model/layer2/ensemble_results.csv   ← 최종 AML 의심 계좌 목록
model/layer2/gnn_weights.pt         ← 학습된 GNN 모델 가중치
model/eda/eda_result.png            ← 데이터 분포 시각화
```

## 자주 발생하는 오류

| 오류 | 원인 | 해결 |
|---|---|---|
| `FileNotFoundError: kftc/` | 데이터 경로 없음 | `model/data/kftc/` 폴더 생성 후 CSV 넣기 |
| `KeyError: 'WD_AC_SN'` | 컬럼명 불일치 | `real_data_loader.py`의 COL_MAP 수정 |
| `UnicodeDecodeError` | 파일 인코딩 문제 | `_load_table()` 의 `encoding="euc-kr"` 또는 `"utf-8"` 로 변경 |
| `CUDA not available` | GPU 없음 | 정상 — CPU로 자동 전환됨 |
